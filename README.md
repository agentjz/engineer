# 智能体工程文档库

这是一套面向“智能体工程”的中文文档库。

它不是代码框架，也不是某个具体项目的实现仓库。
它更像一套可以直接复用的工程骨架，帮助你把软件开发从“靠感觉让 AI 写代码”，升级成“有规约、有契约、有验证、有证据、有演进”的交付方式。

### 1. 智能体工程宪法原则

这部分是给人看的。

它回答的不是“某个页面怎么写”，而是：

- 智能体工程为什么要这样做
- 哪些边界不能被实现细节推翻
- 当规约、代码、测试、文档冲突时，应该先保护什么

每个文档只讲一条原则，尽量写得通俗，但不丢专业性。

## 怎么使用

如果你要开一个新项目，推荐顺序是：

1. 先读“宪法原则”，建立共同的上位规则。
2. 再按当前动作，从“给 AI 看”的目录里选一份最贴近的提示词。
3. 不要一开始就找一份大而全的万能提示词。
4. 让规约、契约、测试、审查、收口各自有明确入口。

一个常见的最小链路是：

1. 进入智能体工程模式
2. 补最小可执行规约
3. 补接口契约
4. 拆成可独立验收任务包
5. 先写失败测试
6. 按最小实现推进
7. 做规约一致性审查
8. 做集成验收与收口

## 这套库强调什么

这套库最核心的几个关键词是：

- 规约
- 契约
- 验收
- 失败测试
- 独立验证
- 交叉验证
- 上下文分层
- 长任务交接
- 多智能体隔离与集成
- 变更演进
- 证据驱动收口

一句话说，就是：

不要让智能体“猜着做”。
要让智能体“按规约做、被验证约束着做、拿证据收口”。
