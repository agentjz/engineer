# P19 反馈信号必须分层且机器可读

## 原则

反馈不能只有一种，也不能只靠人看日志。

反馈必须分层，而且要能被机器直接消费。

## 为什么

Agent 跑得快，问题也扩散得快。

如果所有问题都等到最后一轮人工检查才发现，修复成本会被放大很多倍。

而且多智能体并行时，人根本盯不过来。

## 在智能体工程里的含义

至少要有几层反馈：

- 快反馈：格式、类型、静态检查
- 中反馈：测试、CI、模块级验证
- 慢反馈：集成、E2E、性能、运行时行为

更关键的是，这些反馈不能只是“留一段日志给人看”。
最好是机器一眼就能判断：

- 通过还是失败
- 失败在哪一层
- 该回到哪个任务块修

没有机器可读信号，自动化链路就断了。
