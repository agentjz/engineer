# P20 放手之前先有自动漂移检测

## 原则

你只要准备“放手让智能体自己跑”，就必须先补上自动漂移检测。

## 为什么

人在 loop 里时，很多问题可以靠人工巡检补位。

但一旦你不盯着了：

- spec 过时了没人发现
- 契约变了没人提醒
- 代码沿着旧假设继续长

最后等你回来时，偏差已经积累成一大片。

## 在智能体工程里的含义

放手之前，系统至少要有能力持续检查：

- 规约和代码是不是还一致
- 契约和实现是不是还一致
- 哪些关键规则被改动了
- 哪些地方的验证结果开始失真

卷一里，人发现漂移，人去改。
卷二开始，系统要能先替人发现。

不然“自主执行”只是把失控延后一点点暴露。
