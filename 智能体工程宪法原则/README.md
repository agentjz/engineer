# 智能体工程宪法原则

这一组文档是给人类看方向的根本原则。

它们回答的不是“某个页面怎么写”，而是：

- 智能体工程为什么要这样做
- 哪些边界不能被一时的实现习惯推翻
- 当代码、测试、文档、对话冲突时，应该先保护什么

## 使用规则

1. 这里只写长期有效、跨多个项目都成立的规则。
2. 一次性的技巧、某个框架的小窍门，不要升格成宪法原则。
3. 具体任务怎么做，去看“给 AI 看”的提示词目录。
4. 当局部实现和这里冲突时，以这里为上位约束。
