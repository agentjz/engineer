# P14 用交叉验证查漂移

## 原则

同一份意图，至少要从两个以上角度重新展开，再拿结果互相对照。

## 为什么

智能体最容易出的错，不是完全不会做，而是“做了一版看起来也像那么回事的东西”。

如果只看一轮产物，很难知道它到底是理解对了，还是碰巧写顺了。

交叉验证的作用，就是专门抓这种“表面顺、实际上偏”的漂移。

## 在智能体工程里的含义

最典型的做法是：

1. 先从需求展开成 spec
2. 再从 spec 拆成 task
3. 再从 task 反推 checklist 或验收检查项

如果三份东西互相对得上，说明方向比较稳。
如果中间出现：

- checklist 里冒出 spec 没提过的东西
- task 里漏了 spec 的关键要求
- task 和 checklist 对同一件事说法矛盾

那就是漂移信号。
