# 智能体工程给 AI 看的提示词

这一组文档不是给人类讲概念的，而是给 AI 直接执行的。

每一份提示词只做一件事。
不要指望一份提示词包打天下。

## 现在怎么用

常见顺序一般是这样：

1. 先用 [01-进入智能体工程模式提示词](01.md)
2. 需求不清时，用 [02-补最小可执行规约提示词](02.md)
3. 有协作边界时，用 [03-补接口契约提示词](03.md)
4. 任务太大时，用 [04-拆成可独立验收任务包提示词](04.md)
5. 开发前，用 [05-先写失败测试提示词](05.md)
6. 开始写代码时，用 [06-按最小实现推进提示词](06.md)
7. 做独立检查时，用 [07-做规约一致性审查提示词](07.md)
8. 合并前后，用 [08-做集成验收与收口提示词](08.md)
9. 需求演进时，用 [09-做变更包与规约同步提示词](09.md)
10. 长任务续跑时，用 [10-做长任务交接与续跑提示词](10.md)
11. 多智能体并行时，用 [11-做多智能体并行规划提示词](11.md)
12. 需要把需求写成结构化 spec 时，用 [12-写三维结构化规约提示词](12.md)
13. 需要检查 spec 有没有漂移时，用 [13-做交叉验证提示词](13.md)
14. 需要沉淀项目级上下文时，用 [14-写项目上下文提示词](14.md)
15. 需要拆分角色记忆和任务交接时，用 [15-写角色卡与交接文档提示词](15.md)
16. 需要设计反馈基础设施时，用 [16-设计分层反馈基础设施提示词](16.md)
17. 需要设计协调机制时，用 [17-设计共享状态与协调机制提示词](17.md)
18. 需要在放手前补自动监控时，用 [18-设计自动漂移检测提示词](18.md)
19. 需要把 PRD 拆成边界清楚的子域时，用 [19-把PRD拆成领域树提示词](19.md)
20. 需要回到人来确认意图时，用 [20-做意图对齐复核提示词](20.md)

## 使用规则

1. 一次只拿一份最贴近当前动作的提示词出来用。
2. 如果当前动作变了，就切换到下一份，不要在一份提示词里硬塞所有阶段。
3. 这些提示词默认都服从“宪法原则”目录里的上位规则。
