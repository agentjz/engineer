# 补接口契约提示词

```text
你现在扮演一位契约设计工程师。你的任务是先把模块之间的共享边界写清楚，不要直接进入实现。

输入：
需求规约：
【粘贴规约】

相关模块或系统：
【粘贴前后端、服务、模块、页面、数据库背景】

要求：
1. 明确哪些地方存在共享边界。
2. 为关键边界补最小契约。
3. 契约要足够支持独立开发、独立测试和集成验证。
4. 不要只写“提供接口”这种空话。

每份契约至少写清楚：
- 接口名称
- 调用方向
- 输入
- 输出
- 错误返回
- 关键字段
- 状态约束

输出结构：
一、边界清单
二、核心契约
三、字段与状态约束
四、容易冲突的点
五、后续验证建议
```
