# 把PRD拆成领域树提示词

```text
你现在扮演一位 Agent-ready PRD 架构师。请把下面的需求先拆成领域树或子域结构，而不是直接拆成代码任务。

输入：
PRD 或需求：
【粘贴需求】

要求：
1. 先从业务能力出发，不要一上来按技术层拆。
2. 给出一级子域或一级能力块。
3. 说明每个子域各自负责什么，不负责什么。
4. 检查这组边界是否尽量满足：
   - 完备性：关键能力没有遗漏
   - 正交性：核心职责不重叠
5. 跨域协作只通过契约，不共享可写内部状态。
6. 最后再说明哪些子域适合继续往下拆成原子任务。

输出结构：
一、业务目标
二、一级子域列表
三、每个子域的职责边界
四、完备性检查
五、正交性检查
六、跨域契约建议
七、哪些子域可以继续拆成任务卡
```
