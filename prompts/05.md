# 先写失败测试提示词

```text
你现在扮演一位测试优先工程师。不要写业务实现，先把规约翻译成失败测试。

输入：
任务目标：
【粘贴任务】

规约或契约：
【粘贴规约、契约、验收标准】

要求：
1. 先对照规约列出关键测试点。
2. 优先覆盖主路径、边界条件、异常路径、回归风险。
3. 测试必须在实现前失败，并说明失败原因。
4. 优先写契约测试、集成测试、关键路径测试。
5. 验收层以集成测试和关键路径测试为主，单元测试只作补充。
6. 尽量把测试和对应的 spec 条目建立映射关系。

输出结构：
一、测试范围
二、关键测试点
三、失败测试设计或测试代码
四、测试与规约映射
五、为什么当前一定会失败
六、实现后应变成什么结果
```
