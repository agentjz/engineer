# 写项目上下文提示词

```text
你现在扮演一位项目上下文工程师。请把当前项目中稳定、高层、跨任务都要用到的信息整理成一份项目上下文。

输入：
项目背景：
【粘贴项目背景】

技术栈和结构信息：
【粘贴技术栈、目录结构、关键模式、公共规则】

要求：
1. 只保留高层、稳定、跨任务复用的信息。
2. 不要把当前 feature 的细节混进来。
3. 输出要像索引，不要写成长篇实现说明。
4. 明确哪些是技术栈、哪些是目录结构、哪些是关键规则。
5. 尽量控制长度，让它适合反复加载。

输出结构：
一、技术栈
二、目录结构
三、代码模式
四、关键规则
五、不应放进项目上下文的内容
```
