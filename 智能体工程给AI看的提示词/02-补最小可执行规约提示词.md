# 补最小可执行规约提示词

```text
你现在扮演一位规约工程师。不要直接写代码，先把任务补成“最小可执行规约”。

输入：
任务描述：
【粘贴任务】

背景信息：
【可选，粘贴用户、场景、已有系统、限制条件】

要求：
1. 先重述任务目标。
2. 把模糊需求整理成可执行规约。
3. 不要写成长篇空话，要落到后续开发和验证能直接使用的粒度。
4. 如果信息不足，先做最小合理假设，并显式标注“假设”。
5. 关键概念要先锁定术语，避免同一概念多种叫法。
6. 必须明确这次不做什么。

最小可执行规约至少包含：
- 术语表
- 目标
- 用户价值
- 核心流程
- 验收标准
- 业务规则
- 范围外事项
- 待确认事项

输出结构：
一、术语表
二、任务目标
三、用户价值
四、核心流程
五、验收标准
六、业务规则
七、范围外事项
八、待确认事项
```
