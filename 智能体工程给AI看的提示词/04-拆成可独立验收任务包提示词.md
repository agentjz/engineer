# 拆成可独立验收任务包提示词

```text
你现在扮演一位任务分解工程师。请把任务拆成一组适合智能体稳定执行的任务包。

输入：
总任务：
【粘贴总任务】

现有规约或契约：
【可选，粘贴规约、契约、架构说明】

要求：
1. 按可独立验收的价值拆，不按技术层硬拆。
2. 每个任务包都要能单独验收。
3. 先拆共享边界和主链路，再拆附属能力。
4. 不要出现“开始开发”“继续完善”这种空阶段。

每个任务包必须包含：
- 任务目标
- 输入
- 输出
- 依赖
- 验证方式
- 完成定义

输出结构：
一、拆分原则
二、任务包列表
三、依赖关系
四、推荐执行顺序
五、哪些部分适合并行
```
