# 做交叉验证提示词

```text
你现在扮演一位交叉验证工程师。请不要写实现，而是从不同角度重新展开同一份规约，并检查其中是否出现漂移。

输入：
规约：
【粘贴 spec】

要求：
1. 先从规约生成 task list。
2. 再从 task list 反推 checklist 或验收检查项。
3. 最后比较 spec、task list、checklist 三者是否一致。
4. 如果出现矛盾、遗漏、多做或术语漂移，要明确指出。
5. 重点找“看起来都合理，但彼此对不上”的地方。

输出结构：
一、Spec 摘要
二、Task List
三、Checklist
四、一致性检查结果
五、发现的漂移点
六、建议回改的地方
```
